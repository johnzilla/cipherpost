<technical_decisions>

## AD-1 — Prompter trait signature (RESOLVED: Option A, pre-render in caller)

**Choice:** Pre-render in caller (`run_receive`). The trait gains ONE new parameter: `preview_subblock: Option<&str>`. `TtyPrompter` emits it between `Size:` and `TTL:` when `Some`; test prompters (`AutoConfirmPrompter`, `DeclinePrompter`) ignore it.

**Rationale:**
- `TtyPrompter` stays ignorant of `x509-parser` and `preview::render_x509_preview` — keeps the Prompter trait's concern as "render text + read confirmation," not "parse typed material."
- Test prompters need only a single `_preview_subblock: Option<&str>` parameter added — no crosscut concern (they still ignore all args).
- `run_receive` already matches on `envelope.material` at step 8 to pick the correct `as_*_bytes()` accessor — rendering the preview there is one extra line in the same match arm.
- Symmetric with Phase 7: `run_receive` will add arms for PgpKey / SshKey and call `preview::render_pgp_preview` / `render_ssh_preview` — same pattern applies.

**Rejected Option B** (render inside impl): would force `TtyPrompter` to depend on `preview` and the `MaterialVariant` tag enum, inflating both the trait signature (needs variant + bytes rather than a pre-rendered Option<&str>) and the TtyPrompter impl (one parse call it has no other reason to know about).

## AD-3 — run_send signature (RESOLVED: Option A, add material_variant param)

**Choice:** `run_send` gains `material_variant: MaterialVariant` parameter. The caller (`main.rs::dispatch`) passes the parsed clap value directly. Inside `run_send`, the existing `MaterialSource::Bytes|Stdin|File` stays unchanged; dispatch happens via:

```rust
let plaintext_bytes = read_material(material_source)?;
let material = match material_variant {
    MaterialVariant::GenericSecret => payload::ingest::generic_secret(plaintext_bytes.to_vec())?,
    MaterialVariant::X509Cert => payload::ingest::x509_cert(&plaintext_bytes)?,
    MaterialVariant::PgpKey | MaterialVariant::SshKey => {
        return Err(Error::NotImplemented { phase: 7 });
    }
};
```

**Rationale:**
- Preserves `MaterialSource` unchanged — every existing Phase 2 / Phase 5 integration test (`phase2_self_round_trip`, `phase2_share_round_trip`, `phase2_tamper_aborts_before_decrypt`, `phase2_size_cap`, `phase2_expired_share`, `phase2_idempotent_re_receive`, `pass09_scripted_roundtrip`) passes `MaterialSource::Bytes(...)` or `MaterialSource::Stdin`. Adding a single `MaterialVariant::GenericSecret` arg keeps them all compiling with a one-line change per test.
- `run_send` keeps end-to-end ownership of the send pipeline; `main.rs` dispatch stays thin.

**Rejected Option B** (pre-constructed Material): would refactor every existing round-trip test call site to build `Material::GenericSecret { bytes }` manually, doubling the migration burden for a single-variant-call simplification.

## OQ-1 — `--armor` on GenericSecret (RESOLVED: reject)

**Choice:** `cipherpost receive <uri> --armor` on a non-X509Cert share returns `Error::Config("--armor requires --material x509-cert")` with exit 1 (`Config` already maps to exit 1 via the `_ => 1` default).

**Rationale:** Silent-ignore invites user surprise (user thinks they got PEM, got raw bytes, grep-for-`-----BEGIN` fails, debugs for 20 minutes). Reject at dispatch with a specific reason.

## OQ-3 — DN rendering order (RESOLVED: OpenSSL-forward via Display impl)

Resolved in Plan 02. This plan just notes the outcome: banner shows `Subject: CN=leaf, O=Example, C=US` style (matches `openssl x509 -noout -subject`), NOT strict RFC 4514 backward. SPEC.md update in Plan 04 documents this.

</technical_decisions>

    /// Hand-rolled (no new dep): `-----BEGIN CERTIFICATE-----\n` + 64-char-wrapped
    /// base64-STANDARD + `\n-----END CERTIFICATE-----\n`.
    fn pem_armor_certificate(der: &[u8]) -> Vec<u8> {
        use base64::Engine;
        let encoded = base64::engine::general_purpose::STANDARD.encode(der);
        let mut out = String::with_capacity(encoded.len() + 80);
        out.push_str("-----BEGIN CERTIFICATE-----\n");
        for chunk in encoded.as_bytes().chunks(64) {
            out.push_str(std::str::from_utf8(chunk).expect("base64 output is ASCII"));
            out.push('\n');
        }
        out.push_str("-----END CERTIFICATE-----\n");
        out.into_bytes()
    }
