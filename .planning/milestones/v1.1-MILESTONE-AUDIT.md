## Tech Debt (Non-Critical, Carried Forward)

### Wire-budget escape hatch (Phase 6/7/8 cross-cutting)

Realistic X.509 (~234+ B DER), PGP (variable), SSH (variable), and pin+burn-composed payloads exceed the 1000-byte PKARR BEP44 ceiling. Phase 6 chose Option A (D-P7-03 inheritance): ship `#[ignore]`'d round-trips with positive `Error::WireBudgetExceeded` clean-surface pins. Documented in SPEC.md §Pitfall #22 with the measured 5123-byte composite. Deferred to **v1.2+** as architecturally orthogonal to v1.1's PRD-closure scope. Candidate fixes (per 06-VERIFICATION.md): two-tier storage / chunking / out-of-band payload + inline hash commit.

### Real-DHT release-acceptance gate (Phase 9)

The cross-identity Mainline-DHT round-trip test exists, compiles, and is properly triple-gated (`#[cfg(feature = "real-dht-e2e")]` + `#[ignore]` + `#[serial]`). CI never runs `--features real-dht-e2e` (D-P9-D2 + Pitfall #29). Per-release manual execution via `RELEASE-CHECKLIST.md` is the only gate. **Not a milestone-close blocker** — phase goal is "test EXISTS, COMPILES, properly gated" per Phase 9 VERIFICATION rationale.

### Toolchain divergence (project-wide)

`rust-toolchain.toml` pins **1.88**; `.github/workflows/ci.yml` pins clippy at **1.85** (`dtolnay/rust-toolchain@1.85`). clippy 1.88 enforces `uninlined_format_args` as a default lint, surfacing 65+ instances across `src/` + `tests/`. clippy 1.85 (CI's gate) is silent. CI is the source-of-truth release gate per CLAUDE.md's MSRV-1.85 statement. The specifically-flagged `build.rs:17` instance was fixed in commit `e45347b` (chore: inline `{sha}` format arg). Reconciliation deferred to a v1.2 maintenance pass.

### Code-review advisories (non-blocking)

- **WR-01 (`tests/real_dht_e2e.rs:153`):** propagation-wait should resolve via `bob_transport`, not `alice_transport`, to bypass local pkarr cache. Advisory only — does not affect release-acceptance correctness; manual real-DHT operator can observe directly.
- **WR-02 (`MockTransport::publish` seq):** does not bump `seq`, leaving a latent concurrent-write data-loss window if a future code path races `publish ↔ publish_receipt` on the same keypair. Dormant in v1.1; matters only at future composition.
